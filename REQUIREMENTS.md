# 個人ポータルサイト - 要件定義書

## 1. プロジェクト概要

### プロジェクト名

個人用ポータルサイト (My Personal Portal)

### 開発期間

2025 年 10 月 〜 2025 年 12 月（予定）

### 開発者

koki.ishikawa

### 目的

日常的に使用する情報やツールを一箇所に集約し、効率的な情報管理を実現

### 対象ユーザー

開発者本人（個人利用）

---

## 2. 機能要件

### Phase 1: 必須機能（〜2025 年 12 月）

#### 1. ホーム画面

**1-1. カレンダー表示**

- 月次カレンダー
- 今日の日付のハイライト
- 日付クリックで詳細表示（将来拡張）

**1-2. タスク管理**

- タスクの追加・編集・削除
- 完了/未完了の切り替え（D&D）
- タスク一覧表示
- 優先度設定（D&D）

**1-3. 今日の概要**

- 今日の日付・曜日
- タスク進捗状況
- 天気情報の要約
- 次の予定

#### 2. よく使うサイトリンク集

- ブックマークの登録・編集・削除
- カテゴリ別表示
- サイドメニューで表示

#### 3. クイック検索バー

- 複数検索エンジン対応
  - Google
  - YouTube
  - AWS Docs
  - Udemy
- キーボードショートカット（Ctrl+K）

#### 4. 天気情報

- 現在地の天気取得
- 気温・降水確率表示
- 1 日 1 回自動更新

#### 5. メモ機能

- メモの作成・編集・削除
- 一覧表示
- 作成日時・更新日時の記録

---

### Phase 2: 拡張機能（2026 年 1 月〜）

#### 6. 推しブログ更新情報

- ブログ URL の登録
- 定期的な更新チェック（15 分ごと）
- 新着記事の通知
- 既読/未読管理

#### 7. 習慣トラッカー

- 習慣の登録・編集・削除
- 日々の実行記録
- 週間表示（7 日分）
- 連続記録日数の表示

#### 8. ダークモード

- ライト/ダークモード切り替え
- 設定の永続化
- 自動切り替え機能（時間帯指定）

## 3. 技術スタック

### フロントエンド

- Next.js 14/15（App Router）
- React 18+
- Tailwind CSS
- TypeScript

### バックエンド

- Python 3.11+
- Django 5.x
- Django REST Framework
- Celery + Redis（定期実行）
- BeautifulSoup4（スクレイピング）

### データベース

- PostgreSQL 16

### インフラ

- **Vercel**（フロントエンドホスティング）
- **AWS EC2**（バックエンド API）
  - インスタンスタイプ: t3.micro
- **AWS RDS**（PostgreSQL）
  - インスタンスタイプ: db.t4g.micro
- **AWS Lambda**（定期実行タスク）
- **AWS Route 53**（DNS 管理）

### 開発ツール

- Git / GitHub
- VS Code
- Postman（API テスト）
- pgAdmin（DB 管理）

## 7. 制約事項・リスク

### 制約事項

- 個人利用のため認証機能は不要
- 同時アクセスは 1 人のみを想定
- モバイルアプリ化は Phase 3 以降

### 想定リスク

**リスク 1: AWS 利用料金の超過**

- 影響: 予算オーバー
- 対策: CloudWatch アラート設定、自動起動・停止

**リスク 2: スクレイピング対象サイトの仕様変更**

- 影響: 推しブログ機能が動作しなくなる
- 対策: エラー通知機能、複数サイト対応

**リスク 3: 開発期間の遅延**

- 影響: Phase 1 が完成しない
- 対策: 機能を削ってでも MVP を完成させる
